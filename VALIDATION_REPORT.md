## Trust boundaries and untested external paths

### TxLINE

The included showcase is deliberately labeled synthetic. Its SHA-256 Merkle paths verify locally, but those roots are not claimed to exist in the referenced TxLINE PDAs. Live TxLINE snapshots, SSE streams, real proof endpoints, and on-chain validation transactions were not exercised because no TxODDS credentials were available in the sandbox. The gateway and SDK contain the integration seams and require explicit credentials and route configuration.

### Solana program

The Anchor program source, IDL, program-ID alignment, TypeScript PDA derivation, commitment preimage parity, and CPI account selection were checked. The Rust program was not compiled, deployed, or executed because Rust, Solana CLI, and Anchor CLI were unavailable. Before deployment, run `anchor build`, program tests against localnet/devnet, compute-budget profiling with representative TxLINE V3 proofs, and a real raw-CPI return-data test.

### Browser automation

The studio has jsdom interaction tests, deterministic renderer tests, a successful production bundle, and HTTP preview smoke tests. Full Playwright/Cypress browser automation was not run in this sandbox. Canonical visual and audio bytes are validated independently of the live browser renderer.

## Security checks

- No credentials, authorization tokens, or private package-registry URLs are present in the source archive.
- `.env` files, dependency trees, build products, Turbo caches, and coverage output are excluded from the release ZIP.
- TxLINE secrets are accepted only by the gateway; no `VITE_*` credential path exists.
- The included Anchor program allowlists the published TxLINE devnet and mainnet program IDs and checks root-account ownership before raw CPI.
- Synthetic proof status is explicit in fixture data, API responses, and UI labels.
